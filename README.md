# TableCreator

Веб-приложение на Flask для генерации SQL-скрипта создания таблиц PostgreSQL из настроечного файла (`tables_config`).

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
flask --app app run --port 8080
```

Откройте `http://127.0.0.1:8080`, загрузите конфигурационный файл и получите SQL + скачивание `tables.sql`.

## Поддерживаемые форматы

- `.xlsx` / `.xlsm` (лист `tables_config` — v1-формат, `tables_config_v2` — v2-формат, либо единственный лист)
- `.json`

## Структура Excel (v1-формат, лист `tables_config`)

Каждая таблица описывается вертикальным блоком строк. Первая строка блока — имя таблицы, далее строки-атрибуты:

| Строка (столбец A)         | Обязательная | Описание                                              |
|----------------------------|:---:|-------------------------------------------------------|
| Наименование таблицы       | ✔   | Имя таблицы в БД; фактическое значение — в столбце B+ |
| Код колонки в БД           | ✔   | Коды колонок (по одному на столбец, начиная с B)      |
| Тип                        | ✔   | Тип данных PostgreSQL                                 |
| Размерность                |     | Размерность, например `50` или `255`                  |
| Описание                   |     | Отображаемое наименование колонки                     |
| Обязательность             |     | `да` — NOT NULL; иначе допускает NULL                 |
| Уникальность               |     | `да` — UNIQUE                                         |
| Первичный ключ             |     | `да` — PRIMARY KEY                                    |
| Внешний ключ               |     | Ссылка в формате `table(column)`                      |
| Значение по умолчанию      |     | Значение DEFAULT для колонки                          |

## Структура Excel (v2-формат, лист `tables_config_v2`)

Таблицы располагаются **горизонтальными блоками** на одном листе: несколько блоков стоят рядом, разделённых пустым столбцом.

**Строка 1** — имя таблицы (в первой ячейке блока).  
**Строка 2** — заголовки атрибутов колонок.  
**Строки 3+** — по одной строке на каждую колонку.

| Заголовок (строка 2)      | Обязательный | Описание                                      |
|---------------------------|:---:|-----------------------------------------------|
| Описание                  | ✔   | Начало блока; отображаемое наименование колонки|
| Код колонки в БД          | ✔   | Код колонки в БД                              |
| Тип                       | ✔   | Тип данных PostgreSQL                         |
| Размерность               |     | Размерность, например `50` или `255`          |
| Обязательность            |     | `да` — NOT NULL; иначе допускает NULL         |
| Уникальность              |     | `да` — UNIQUE                                 |
| Первичный ключ            |     | `да` — PRIMARY KEY                            |
| Внешний ключ              |     | Ссылка в формате `table(column)`              |
| Значение по умолчанию     |     | Значение DEFAULT для колонки                  |

> **Примечание.** Блок определяется по наличию заголовка `Описание` в строке 2. Столбец `Описание` является обязательным признаком начала блока; сами данные в нём необязательны.

Пример расположения двух таблиц на листе:

```
| Описание      | Код колонки в БД | Тип      | … | | Описание | Код колонки в БД | Тип     | … |
| users                                          | |  orders                                    |
| Идентификатор | id               | bigserial | … | | …        | …                | …       | … |
| Имя           | name             | varchar   | … | | …        | …                | …       | … |
```

## Структура JSON

```json
{
  "tables_config": [
    {
      "table_name": "users",
      "columns": [
        {
          "column_code": "id",
          "type": "bigserial",
          "nullable": false,
          "unique": false,
          "primary_key": true
        },
        {
          "column_code": "email",
          "type": "varchar",
          "size": "255",
          "nullable": false,
          "unique": true,
          "column_name": "Электронная почта"
        },
        {
          "column_code": "role_id",
          "type": "bigint",
          "nullable": true,
          "foreign_key": "roles(id)"
        }
      ]
    }
  ]
}
```

Поле `tables_config` может быть либо **списком** объектов (как выше), либо **словарём** `{ "table_name": { columns } }`.

| Поле таблицы  | Тип       | Описание                                           |
|---------------|-----------|----------------------------------------------------|
| `table_name`  | string    | Наименование таблицы (обязательное)                |

| Поле колонки  | Тип       | Описание                                           |
|---------------|-----------|----------------------------------------------------|
| `column_code` | string    | Код колонки в БД (обязательное)                    |
| `type`        | string    | Тип данных PostgreSQL (обязательное)               |
| `size`        | string    | Размерность, например `"255"`                      |
| `nullable`    | bool      | `true` — допускает NULL (по умолчанию `true`)      |
| `unique`      | bool      | `true` — уникальное значение (по умолчанию `false`)|
| `primary_key` | bool      | `true` — первичный ключ (по умолчанию `false`)     |
| `foreign_key` | string    | Ссылка в формате `table(column)`                   |
| `default`     | string    | Значение по умолчанию                              |
| `column_name` | string    | Отображаемое наименование колонки                  |

## Что проверяется

- Дубликаты таблиц
- Дубликаты колонок внутри таблицы
- Имена таблиц/колонок/FK в формате PostgreSQL (`[A-Za-z_][A-Za-z0-9_]*`)
- Использование зарезервированных слов PostgreSQL
