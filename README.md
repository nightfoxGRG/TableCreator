# TableCreator

Веб-приложение на Flask для работы со структурой таблиц PostgreSQL.

Приложение включает два инструмента:
1. **Генератор конфигурационного файла** — определяет структуру таблицы по загруженному файлу данных и формирует готовый конфигурационный файл.
2. **Генератор SQL** — создаёт SQL-скрипт `CREATE TABLE` по конфигурационному файлу или JSON-описанию таблиц.

---

## Возможности

### 🔍 Генератор конфигурационного файла (`/`)

- Принимает файл с данными в форматах `.xlsx`, `.xlsm`, `.csv`.
- Первая строка файла — заголовки колонок (названия на любом языке, включая русский).
- Автоматически **переводит названия колонок на английский** через сервис LibreTranslate.
- **Определяет тип PostgreSQL** для каждой колонки по содержимому данных:
  - `boolean` — если все значения `true`/`false`
  - `date` — если все значения в формате `YYYY-MM-DD`
  - `timestamp` — если встречаются значения с временем `YYYY-MM-DD HH:MM...`
  - `integer` / `bigint` — целые числа (до/свыше 2 147 483 647)
  - `numeric(p,s)` — дробные числа с вычисленными точностью и масштабом
  - `varchar(N)` — строки до 255 символов (длина округляется вверх до кратного 50)
  - `text` — строки длиннее 255 символов или колонки без данных
- Формирует **имя SQL-идентификатора колонки** из заголовка: переводит, удаляет недопустимые символы, приводит к `snake_case`.
- Генерирует конфигурационный файл в формате **`tables_config_v2`** (`.xlsm`) на основе шаблона `TablesConfig.xlsm`.
- Предлагает скачать сформированный файл через модальное окно.
- При недоступности сервиса перевода выводит понятное сообщение об ошибке.

### ⚙️ Генератор SQL (`/sql`)

- Принимает конфигурационный файл в форматах `.xlsx`, `.xlsm`, `.json`.
- Парсит описание таблиц в двух форматах Excel:
  - **v1** (лист `tables_config`) — вертикальные блоки
  - **v2** (лист `tables_config_v2`) — горизонтальные блоки
- Поддерживает **JSON-описание** таблиц (формат `tables_config`).
- Генерирует **корректный SQL-скрипт** `CREATE TABLE` для PostgreSQL:
  - Типы данных с размерностью: `varchar(N)`, `numeric(p,s)` и др.
  - Ограничения: `NOT NULL`, `UNIQUE`, `PRIMARY KEY`, `DEFAULT`, `REFERENCES`
  - Значения `DEFAULT` форматируются согласно типу (числа — без кавычек, строки — в кавычках, SQL-константы `NULL`, `TRUE`, `FALSE`, `now()` и др. — как есть)
  - Выравнивание колонок и комментарии с отображаемым именем
- **Валидирует** конфигурацию перед генерацией:
  - Дубликаты таблиц и колонок
  - Имена таблиц/колонок/внешних ключей в формате PostgreSQL (`[A-Za-z_][A-Za-z0-9_]*`)
  - Использование зарезервированных слов PostgreSQL
  - Корректность ссылок внешних ключей (формат `table(column)`)
  - Корректность значений `DEFAULT` для типа колонки
- Отображает SQL прямо на странице.
- Позволяет **скачать шаблон** `TablesConfig.xlsm` для ручного заполнения.
- Позволяет **скачать сгенерированный SQL** как файл `tables.sql`.

### 🖥️ Режимы запуска

- **Серверный режим** — стандартный запуск через Flask (фон основной области — белый, бейдж режима — розовый).
- **Локальный режим** — запуск из собранного исполняемого файла PyInstaller (фон — нежно-голубой, бейдж режима — голубой).

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate       # macOS / Linux
# .venv\Scripts\activate        # Windows
pip install -r requirements.txt
flask --app app run --port 8080
```

Откройте `http://127.0.0.1:8080`.

### Сервис перевода (LibreTranslate)

Для автоматического перевода заголовков колонок с русского на английский требуется запущенный экземпляр [LibreTranslate](https://libretranslate.com/).

URL сервиса задаётся в файле `config.toml`:

```toml
[translation]
libretranslate_url = "http://127.0.0.1:50001"
```

Запуск через Docker Compose:

```bash
docker compose -f docker/libretranslate.yml up -d
```

Приложение можно собрать в самостоятельный исполняемый файл для macOS и Windows с помощью [PyInstaller](https://pyinstaller.org/):

```bash
pyinstaller DataPipelinePro.spec
```

Собранный файл не требует установки Python и запускается в **локальном режиме**.

## Поддерживаемые форматы конфигурации

### Excel v1 (лист `tables_config`)

Каждая таблица — вертикальный блок. Первая строка блока — имя таблицы, далее строки-атрибуты:

| Строка (столбец A)         | Обязательная | Описание                                              |
|----------------------------|:---:|-------------------------------------------------------|
| Наименование таблицы       | ✔   | Имя таблицы в БД; значение — в столбцах B+            |
| Код колонки в БД           | ✔   | Коды колонок (по одному на столбец, начиная с B)      |
| Тип                        | ✔   | Тип данных PostgreSQL                                 |
| Размерность                |     | Например `50` или `10,2`                              |
| Описание                   |     | Отображаемое наименование колонки                     |
| Обязательность             |     | `да` — NOT NULL                                       |
| Уникальность               |     | `да` — UNIQUE                                         |
| Первичный ключ             |     | `да` — PRIMARY KEY                                    |
| Внешний ключ               |     | Ссылка в формате `table(column)`                      |
| Значение по умолчанию      |     | Значение DEFAULT для колонки                          |

### Excel v2 (лист `tables_config_v2`)

Таблицы — горизонтальные блоки, разделённые пустым столбцом. Каждая колонка — отдельная строка.

**Строка 1** — имя таблицы. **Строка 2** — заголовки атрибутов. **Строки 3+** — данные колонок.

| Заголовок (строка 2)      | Обязательный | Описание                                      |
|---------------------------|:---:|-----------------------------------------------|
| Описание                  | ✔   | Признак начала блока; наименование колонки    |
| Код колонки в БД          | ✔   | Код колонки в БД                              |
| Тип                       | ✔   | Тип данных PostgreSQL                         |
| Размерность               |     | Например `50` или `10,2`                      |
| Обязательность            |     | `да` — NOT NULL                               |
| Уникальность              |     | `да` — UNIQUE                                 |
| Первичный ключ            |     | `да` — PRIMARY KEY                            |
| Внешний ключ              |     | Ссылка в формате `table(column)`              |
| Значение по умолчанию     |     | Значение DEFAULT для колонки                  |

### JSON

```json
{
  "tables_config": [
    {
      "table_name": "users",
      "columns": [
        {
          "column_code": "id",
          "type": "bigserial",
          "nullable": false,
          "primary_key": true
        },
        {
          "column_code": "email",
          "type": "varchar",
          "size": "255",
          "nullable": false,
          "unique": true,
          "column_name": "Электронная почта"
        },
        {
          "column_code": "role_id",
          "type": "bigint",
          "nullable": true,
          "foreign_key": "roles(id)"
        }
      ]
    }
  ]
}
```

Поле `tables_config` — список объектов или словарь `{ "table_name": { columns } }`.

| Поле колонки  | Тип    | Описание                                           |
|---------------|--------|----------------------------------------------------|
| `column_code` | string | Код колонки в БД (обязательное)                    |
| `type`        | string | Тип данных PostgreSQL (обязательное)               |
| `size`        | string | Размерность, например `"255"` или `"10,2"`         |
| `nullable`    | bool   | `true` — допускает NULL (по умолчанию `true`)      |
| `unique`      | bool   | `true` — UNIQUE (по умолчанию `false`)             |
| `primary_key` | bool   | `true` — PRIMARY KEY (по умолчанию `false`)        |
| `foreign_key` | string | Ссылка в формате `table(column)`                   |
| `default`     | string | Значение по умолчанию                              |
| `column_name` | string | Отображаемое наименование колонки                  |

## Автоопределение типов PostgreSQL

| Условие                                      | Тип PostgreSQL       |
|----------------------------------------------|----------------------|
| Все значения `true` / `false`                | `boolean`            |
| Все значения в формате `YYYY-MM-DD`          | `date`               |
| Есть значения с временем `YYYY-MM-DD HH:MM…` | `timestamp`          |
| Целые числа ≤ 2 147 483 647                  | `integer`            |
| Целые числа > 2 147 483 647                  | `bigint`             |
| Дробные числа                                | `numeric(p,s)`       |
| Строки ≤ 255 символов                        | `varchar(N)` (N кратно 50, минимум 50) |
| Строки > 255 символов или пустая колонка     | `text`               |
